Code Review - Code of Conducts

  • Provide code quality together - Team Work
  • Learn and improve together
  • Code Review is a discussion process to reach a consensus.
  • Provide best practices, APIs, links to resources i.e wiki, stack overflow, etc.
  • Collaborate with teammate/s to move Code Review forward via. g-chat, or g-meet, etc.
  • Provide constructive comments, "This will improve ..." or "According to spec... " or "Java Coding Standards ...", "Sonar findings" etc.
  • Collaborate with asking questions and provide answers to understand each-other
  • Review big PR/s with pause breaks which helps you to recharge/rehydrate, and to concentrate more
